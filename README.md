## ‚öôÔ∏è Complete Configuration Guide

## üîÑ How the Smart Workflow Works

### 1. **Detection Phase**

- Monitors saves of `.ts` and `.tsx` files
- Scans for `interface` and `type` declarations
- Extracts complete definitions including multi-line types

### 2. **Extraction Phase**

```typescript
// Finds and extracts this entire block:
interface ComplexInterface {
  id: string;
  metadata: {
    created: Date;
    updated?: Date;
  };
  permissions: Array<{
    role: string;
    actions: string[];
  }>;
}
```

### 3. **Organization Phase**

- Merges with existing types file
- Sorts types alphabetically (types first, then interfaces)
- Handles duplicate interface merging intelligently

### 4. **Source Cleanup Phase** (if enabled)

- Removes interface definition from original file
- Calculates optimal import path automatically
- Updates or creates import statement
- Preserves existing imports and adds new ones

### 5. **Statistics Tracking**

- **Types Added**: Count of definitions added to types file
- **Types Removed**: Count removed during cleanup
- **Interfaces Extracted**: Count extracted from source files
- **Lines Removed**: Actual lines removed from source files

## üîç Troubleshooting & FAQ

### ‚ùì **Extension Not Working?**

- ‚úÖ Check status bar for üßπ icon (should be visible)
- ‚úÖ Verify types file exists at configured path
- ‚úÖ Ensure you're saving `.ts` or `.tsx` files
- ‚úÖ Check VS Code/Cursor output panel for errors
- ‚úÖ Verify extension is enabled (not showing ‚ùå)

### ‚ùì **Types File Not Found?**

```bash
# Create types file if missing
mkdir -p types
touch types/Types.d.ts

# Or configure different path in settings
{
  "typesCleanup.typesFileName": "your-custom-path/types.d.ts"
}
```

### ‚ùì **Interfaces Not Being Extracted?**

- ‚úÖ Verify proper TypeScript syntax
- ‚úÖ Check that definitions start with `interface` or `type`
- ‚úÖ Ensure files have `.ts` or `.tsx` extensions
- ‚úÖ Confirm extension is enabled (green status bar)
- ‚úÖ Test extraction: Command Palette ‚Üí "Types Cleanup: Test Type Extraction"

### ‚ùì **Source Cleanup Not Working?**

- ‚úÖ Check `typesCleanup.cleanupSourceFiles` is `true`
- ‚úÖ Verify the interface was actually extracted
- ‚úÖ Look for any TypeScript syntax errors in the file
- ‚úÖ Check console logs for detailed debug information

### ‚ùì **Import Paths Wrong?**

The extension automatically calculates relative paths. If imports are incorrect:

- ‚úÖ Verify your types file is in the expected location
- ‚úÖ Check workspace root is correctly detected
- ‚úÖ Manually adjust the generated import if needed

### ‚ùì **Cleanup Removing Used Types?**

- ‚úÖ Wait for cleanup delay (default: 2 seconds)
- ‚úÖ Check if types are used in comments or strings (not detected)
- ‚úÖ Disable auto-cleanup: `"typesCleanup.enableAutoCleanup": false`
- ‚úÖ Use manual cleanup for more control
